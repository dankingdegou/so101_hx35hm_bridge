# HX-35HM + SO101 装配姿态与归零建议（含 rest 姿态解释）

你提到的担心是对的：**如果 SO101 是按“rest（折叠）姿态”装配/定义零位的，那么“把 6 个舵机全部打到中位(pos=500)再装配”很可能不对**，因为：

- HX-35HM 的“中位”是舵机角度的中位（默认 `pos=500 ≈ 120°`），不是机器人关节角的 `0 rad`，更不是 MoveIt 的 `rest` 关节角集合。
- 机器人装配时真正重要的是：**你选定的参考姿态（零姿态/折叠 rest/其它）下，每个关节的机械对齐与软件角度定义一致**。

下面先给出“仓库里 rest 是什么”的证据，再给一份更稳妥、更全面的装配流程。

---

## 1. so101 仓库里确实定义了 `rest`（折叠）姿态

在 MoveIt 的 SRDF 里，这个工程给机械臂（`manipulator`）定义了多个命名姿态，包括 `zero`、`rest`、`extended`：

- 文件：`~/ros2_ws/src/so101-ros-physical-ai/so101_moveit_config/config/so101_arm.srdf`

其中 `rest` 的关节角（单位 rad）是：

- `shoulder_pan = 0`
- `shoulder_lift = -1.57`
- `elbow_flex = 1.57`
- `wrist_flex = 0.75`
- `wrist_roll = 0`

SRDF 的注释也明确说这是用来表示类似“folded arms（折叠）”这种命名状态的。

重要点：**这只能说明 MoveIt 里有一个“折叠 rest 状态”的定义，并不能 100% 证明机械装配就是按它来的**；但它是你在软件侧最明确、最可复现的一套“折叠参考姿态”。

---

## 2. 结论：装配时不应该“所有舵机都去同一个 pos”

### 2.1 “全中位装配”什么时候成立

只有当你明确选择的装配参考姿态满足：

- 该姿态下每个关节期望角度都对应“舵机物理中位”（或者你人为定义的同一个舵机 pos），并且
- 每个关节的连杆/舵盘安装孔位允许这样对齐

这在实际机械臂上很少成立。

### 2.2 更正确的思路

装配要做的是：

1. 先选一个“参考姿态”（推荐二选一：`zero` 或 `rest`）
2. 为每个关节确定：
   - 方向：`+1 / -1`
   - 0 rad 对应的舵机位置：`joint_zero_pos`（每个关节通常不同）
3. 装配时把“每个舵机”打到它在参考姿态下的目标位置，再装上对应连杆/舵盘

---

## 3. 推荐的装配参考姿态（选一种）

### 选项 A：按 `zero` 姿态装配（所有关节 0 rad）

优点：

- 软件上最直观，`0 rad` 就是你装配对齐的姿态
- 之后 MoveIt/控制里“回零”很清晰

缺点：

- 机械结构上不一定是最“收拢/安全”的姿态，装配时可能占空间、易干涉

### 选项 B：按 `rest` 姿态装配（折叠）

优点：

- 常用于“收拢、方便搬运”的安全姿态
- 如果你的结构在折叠时孔位更好对齐，会更省事

缺点：

- 软件上 `0 rad` 不再等于“装配姿态”，你需要在桥接/配置里明确映射

---

## 4. 你现在的 bridge 已经支持“按任意参考姿态装配”所需的参数

bridge 文件：

- `~/ros2_ws/src/so101_hx35hm_bridge/so101_hx35hm_bridge/bridge_node.py`

已支持（按 `joint_names` 顺序对齐）：

- `joint_directions`: 每关节 `1` 或 `-1`（决定正方向）
- `joint_zero_positions`: 每关节 “0 rad 对应的舵机 pos”
- `servo_zero_pos`: 全局默认零位（你没单独填 `joint_zero_positions` 时才用）

因此装配时你可以：

1. 先把舵机打到某个“装配参考姿态”的目标 pos
2. 装好以后，再通过 `joint_zero_positions` 把“0 rad 的位置”精确标定到你想要的姿态

## 5. 装配流程（更全面、更稳）

### 5.1 装配前的硬件安全建议

- 供电先保守：限流电源优先，或者串一个保险/限流模块
- 第一次上电别带负载大动作：先拆下连杆/末端，让单舵机轻载测试方向和范围
- 舵机和结构避免硬顶：任何“打到端点再装配”的方式都不推荐

### 5.2 先做“单舵机验证”再装整臂

推荐顺序（每个舵机都做一遍）：

1. 单独接一个舵机，确认 ID 正确（`change_servo_id.py`）
2. 用 `continuous_sweep.py` 做小幅扫动（确认无异常、方向记录清楚）
3. 把它停在你希望的装配角附近（不要顶到极限）

### 5.3 选择参考姿态后，计算每个关节的“装配目标 pos”

HX-35HM 默认映射（按说明书常见默认）：

- `pos` 范围 `0..1000` 对应 `0..240°`
- 每 1° 大约 `1000/240 ≈ 4.1667` 个 pos

如果你暂时假设（仅用于“举例说明”）：

- 每个关节方向都是 `+1`
- 每个关节 `0 rad -> pos=500`

那么 SRDF 里 `rest` 对应的“理论 pos”大概是：

- `shoulder_pan 0` -> `500`
- `shoulder_lift -1.57 rad ≈ -90°` -> `500 - 375 ≈ 125`
- `elbow_flex 1.57 rad ≈ +90°` -> `500 + 375 ≈ 875`
- `wrist_flex 0.75 rad ≈ +43°` -> `500 + 179 ≈ 679`
- `wrist_roll 0` -> `500`

注意：**这只是示例**。实际装配里每个关节的方向很可能不同，而且“0 rad 对应的舵机 pos”也不一定是 500。

### 5.4 推荐的装配动作（逐关节）

对每个关节重复以下步骤会更稳：

1. 让该关节的舵机转到“参考姿态下的目标 pos”（不要顶限位）
2. 在舵机保持通电、不可自由旋转的状态下装上舵盘/连杆
3. 确保：
   - 连杆安装没有预应力（不需要硬掰才能装上）
   - 该关节在参考姿态下的机械角确实处于你期望的位置（比如折叠/伸直）
4. 装完一个关节就做一个小幅正反摆动测试（例如 ±10°），确认不会干涉

这比“一次性把 6 个舵机都打到同一个值，然后整臂装完再祈祷不偏”要可靠得多。

### 5.5 装完后：把“软件零位/方向”固化

装完后你要得到两张表（写到文档里备份）：

- `关节名 -> 舵机 ID`（你已经用 `JOINT_ID_MAP` 体现了）
- `关节名 -> 方向(+1/-1) 与 0rad 对应舵机 pos`

然后把它们作为参数传给 `hx35hm_bridge`（用 launch 或命令行 `--ros-args -p ...`）。

---

## 6. “先全中位再装配”怎么用才不坑

“全中位”可以作为 **装配前的安全中间态**：

- 用来让所有舵机远离端点，方便你安装舵盘、插拔、上电检查
- 用来保证第一次装配时没有某个舵机停在危险位置

但它不应该被当成“最终参考姿态”。

更推荐的做法是：

1. 全中位（安全起点）
2. 然后逐关节调到该关节的目标 pos（`zero` 或 `rest`）再装该关节
